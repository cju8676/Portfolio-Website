WebCheckers

Technologies Used:
  • Java
  • FreeMarker
  • CSS
  • JavaScript
  • JUnit
  • Agile/Scrum Methodology
Description:

This is a semester project through SWEN.261 - Intro to Software Engineering at RIT.
Collaborated with four other students with consistent communication through Slack, Trello, and Zoom while merging code through Git/Github.


The application emulates a game of American checkers that can be played between two players across the same connection. More players can chose to spectate the game in real time. Replays of games are also saved and can be viewed once the game is finished.


Sign In

Users are prompted to enter a name that no one else is using and contains at least one alphanumeric character as well as no spaces or abnormal characters.


Main Page

The home page shows the players online that you can play against, current active games between players, and session archived replays from past games that can be viewed.


Gameplay

Games are ran though Ajax get and post routes that are invoked when a situation in the game happens. Given that it is a players turn they can make a move on a valid square. If they make a move they do not want to make, the backup button resets that move if they didn't already submit the turn using the submit turn button. Players can also resign at any time which ends the game and declares the other player the winner.


Viewing Replays

When a user chooses to view a replay of a game, they are given Next and Previous buttons to jump through the game turn by turn. When they are done viewing the replay, they can use the Exit button to bring them back to the Home page.
