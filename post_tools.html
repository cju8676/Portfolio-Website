Tool Management WebApp

Technologies Used:
  • Python
  • React JS
  • PostgreSQL
  • JavaScript
  • Flask
  • Database hosted by RIT
Description:

This is a semester project through CSCI.320 - Principles of Data Management at RIT.
Collaborated with two other students with consistent communication through Slack, Discord, and Zoom while merging code through Git/Github.


View Source Code

This application demonstrates a web version of someone's toolbox. A user is able to add a tool to their toolbox, view a list of available tools other users have made shareable, send a request to borrow that tool, and lend tools to others if they so chose.


Sign In

Upon startup users are prompted to either enter their credentials or create a new account. Passwords are encrypted using SHA-256 encryption upon each login.


Toolbox

Users can see a table full of all the tools they have entered, search for a tool in their toolbox, edit, delete, or create a new tool entry.


Available Tools

When users make a tool shareable, it shows up in the Available tools panel, where users can request to borrow a tool for a certain period of time. When a request is sent, the app recommends other tools to request based on the tools users in the app typically borrow together, similar to Amazon's "Customers also bought".


Incoming/Outgoing Tools

The rest of the tabs include screens for displaying a users requests they have sent, requests they have coming into them, tools they have lent out to others, as well as tools they have borrowed from other users. A tool is marked red if it is overdue its return date.
